Notion+AI-自动生成博客aiagent搭建

发表于2025-12-22|更新于2025-12-23|技术
|浏览量:

嗨，松弟姐妹们！告别码字地狱，用 AI 打造你的 Notion 博客自动分发系统！🚀

有没有觉得写博客就像打一场硬仗？从 Notion 里整理笔记、构思结构、再到吭哧吭哧地把草稿润色成一篇像样的文章……我的天，这活儿真不是一般的耗时间！每次看到自己的干货笔记堆积如山，却没时间转化为优质内容分享出去，那种感觉，一个字——“糙”！

所以，今天咱们就来整一个有点“邪性”但贼实用的玩法：基于 LLM (大语言模型) 的个人知识库自动化分发系统！ 简单来说，就是让 AI 替你把 Notion 里的“毛坯”笔记，一键变成能直接发出去的“精装修”博客文章！是不是很酷！

这套系统，我的目的特纯粹：减少起稿和整理工作，让你的 Notion 笔记自动变身结构化文章，解放你的双手和大脑！

核心玩法概览：

  • 你的大脑基地： Notion (存放你的原始笔记)
  • 幕后大管家： n8n (自动化工作流的魔法棒)
  • 算力发动机： Docker (让 n8n 跑起来的小容器)
  • 智慧大脑： Google Gemini (或者其他 LLM，帮你润色文章)

流程示意图：
Notion 数据拉取 → LLM 生成文稿 → Markdown 格式转换 → 邮件推送/其他分发

咱们会用到 Notion API、Gemini API 和 QQ 邮箱的服务。别担心什么文本转换、API 获取这些“难点”，我都会一步步带你趟平！

第一步：整装待发——环境准备清单

俗话说得好，工欲善其事，必先利其器。咱们先搭好地基！

  1. Docker： 自动化工作流的运行环境，必装！ 如果你是 Windows 用户，推荐再搞个 Docker 图形化界面 (例如 Docker Desktop)，管理起来更丝滑。
  2. Notion： 你的核心知识库，必装！
  3. QQ 邮箱： （或其他支持 SMTP 的邮箱）咱们文章生成后用来推送的。
  4. Google AI Studio： 获取 Gemini API 密钥的地方，当然你也可以换成其他的 LLM 服务，只要能搞到 API 就行。

第二步：给 Notion 配个“小秘书”——数据准备

要让 n8n 和 AI 帮你管理 Notion 笔记，得先给 Notion 开个“后门”并准备好数据。

  1. 创建你的 Notion 数据库：

    • 在 Notion 里新建一个数据库（Database），你可以把它命名为“博客草稿箱”或者“我的文章库”。
    • 这个数据库里至少需要包含以下属性：Status (状态，比如“草稿”、“待发布”、“已发布”) 和 Title (标题)。
    • 小 Tip：Notion AI 现在很强大，你可以直接让它帮你创建包含这些属性的数据库，省心又省力！

  2. 创建 Notion 集成 (Internal Integration)：

    • 这是让外部服务（比如 n8n）能访问你 Notion 数据的“钥匙”。

    • 前往 Notion 官网，登录后创建一个 Internal Integration。

    • 创建完成后，你会得到一个 Internal Integration Secret，这个秘钥务必保存好！ 别轻易泄露给别人。

    • 重要一步： 记得把你创建的 Notion 数据库分享给这个集成，否则它没权限访问你的数据哦！在数据库页面右上角点击 ...，然后选择 Add connections，找到你创建的集成并添加。

第三步：魔法开启——n8n 工作流搭建大揭秘

重头戏来了！咱们要用 n8n 搭建一个完整的自动化工作流。记住我的小提示：每完成一个节点就保存并执行一下，这样有问题也能及时发现！

1. 触发器 (Trigger)：小侦察兵出动！

  • 拖拽一个 Notion Trigger 节点。
  • 配置它监听 Page Updated (页面更新) 事件。
  • 这样，每次你的 Notion 笔记有变动时，n8n 就能立刻感知到。

2. 过滤器 (Filter)：只放行“精锐”！

  • 拖拽一个 If 节点。
  • 配置条件：判断 Notion 笔记的 Status 属性是否等于 Ready to Publish。
  • 只有当你的笔记状态被设置为“待发布”时，工作流才会继续，避免误操作。

3. 数据抓取 (Fetch)：收集情报！

  • 拖拽一个 Notion 节点。
  • 操作选择 Get Many Children，目标是获取当前 Notion 页面内的所有 Blocks (即正文内容)。
  • 这是为了把你的原始笔记内容全部抓取下来，准备交给 AI。

4. 内容清洗 (Clean)：把“毛坯”变“半成品”！

  • 拖拽一个 Code 节点 (JavaScript)。
  • Notion API 返回的内容是复杂的 JSON 格式，AI 不喜欢直接吃这种“生肉”。我们需要用 JavaScript 把这些 Block JSON 提取并拼接成干净的纯文本 Markdown 格式。

  • Prompt (提示词) 是关键！ 把上一步清洗出来的 clean_content 作为输入，然后加上一段“人设”和“要求”明确的 Prompt，告诉 AI 你想要什么风格的文章。

  • AI 生成的文章是纯文本，但为了方便邮件附件传输，我们通常需要把它转换成 Base64 编码的二进制流。

7. 交付 (Delivery)：把文章发出去！

  • 方案 A (推荐)： Send Email 节点 (使用 QQ SMTP)。

    • 配置好你的 QQ 邮箱 SMTP 服务器信息 (需要开启 QQ 邮箱的 SMTP 服务，并获取授权码作为密码)。
    • 设置收件人、主题，并选择将上一步生成的 Markdown 二进制流作为附件发送。

    • 这是最直接的交付方式，你可以在邮件里直接看到生成好的文章。

  • 方案 B (本地)： Write to Disk 节点。

    • 如果你只是想把文章保存到本地文件系统，可以选择这个。前提是你的 Docker 容器挂载了本地目录。

8. 闭环 (Loop)：“已发布”标记！

  • 拖拽一个 Notion 节点。
  • 操作选择 Update Page。
  • 将原始 Notion 笔记的 Status 属性更新为 Published。
  • 这样，你的 Notion 数据库就能清晰地显示哪些文章已经被自动化处理并发布了，实现流程闭环。

大功告成，查收邮件！

当你成功运行整个工作流，并把 Notion 笔记的状态改为 Ready to Publish 后，稍等片刻，去你的收件箱看看，是不是有一封带着 Markdown 附件的邮件躺在那里？那，就是你的 AI 助理帮你生成的博客文章！

一些小提示和未来的展望：

  • 多保存，多测试： 每次添加或修改节点后，一定要保存工作流，并尝试执行一下当前的节点，确保没有错误。
  • 遇到问题不慌： n8n 的日志很详细，配合 AI (没错，就是那个写文章的 AI！) 和官方文档，大部分问题都能解决。当然，你也可以随时来找我交流！
  • 这套架构是通用的： 今天我们用了 Notion、Gemini 和 QQ 邮箱，但这只是冰山一角。未来你可以把 Notion 换成飞书、语雀，把 Gemini 换成 GPT-4、Claude，把 Email 换成 WordPress 自动发帖、或者发布到你的独立博客，原理完全一样！你可以根据自己的需求，灵活替换各个模块。

祝贺你完成构建！ 从此告别繁琐的码字，让 AI 成为你创作路上的得力助手。现在，去尽情探索吧！🚀
